แพลตฟอร์มมาร์เก็ตเพลสสำหรับ **"นักล่า Top Spender ที่มีสินค้าพร้อมส่งทันที"** แต่นำมาปล่อยขายต่อในราคาถูกลงเพื่อดึงทุนคืน โดยระบบจะโฟกัสที่ความโปร่งใส การโชว์หลักฐานใบเสร็จ และการอัปเดตยอดสะสมแบบ Real-time

---

## 💡 แนวคิดหลักของระบบ (Core Concept)

* **ต้องมีสินค้าพร้อมส่งเท่านั้น (No Pre-order):** ผู้ขาย (Top Spender) ต้องซื้อสินค้าจริงจากแบรนด์หลักมาเรียบร้อยแล้ว จึงจะสามารถลงขอบนแพลตฟอร์มได้ เพื่อลดความเสี่ยงจากการเทออเดอร์
* **ลดราคาเพื่อดึงทุนคืน:** สินค้าจะถูกตั้งราคาขายต่ำกว่าป้าย (เช่น ลด 30-50%) เพื่อให้ขายออกได้ไวที่สุด 
* **สิทธิ์ในการสะสมยอด:** ผู้ซื้อยอมรับว่าจะได้เฉพาะ "ตัวสินค้า" ในราคาถูก ส่วน "คะแนนสะสม/สิทธิ์ Top Spender" จะยังคงเป็นของผู้ขายตามใบเสร็จหลัก

---

## 📱 ฟีเจอร์สำคัญที่ต้องมี (Key Features)

### 1. ระบบสำหรับผู้ขาย (Top Spender Dashboard)
* **Verified Stock & Receipt:** ระบบบังคับให้อัปโหลด **ภาพถ่ายคู่กับสินค้าจริง** และ **ใบเสร็จจากแบรนด์หลัก** ก่อนเปิดขาย
* **Live Progress Tracker:** แถบแสดงสถานะยอดซื้อรวมของผู้ขายรายนั้น ๆ เช่น *“ผู้ขายคนนี้มียอดสะสมปัจจุบัน 120,000 บาท จากเป้าหมาย 200,000 บาท”* เมื่อมีคนกดซื้อสินค้า ยอดสะสมนี้จะอัปเดตแบบ Real-time เพื่อสร้างความมั่นใจให้ลูกค้าเห็นว่าเงินที่จ่ายไปช่วยดันผู้ขายจริง ๆ

### 2. ระบบสำหรับผู้ซื้อ (Buyer Dashboard)
* **Flash Sale & Clearance Price:** หน้าแรกที่รวมดีลสินค้าแท้ราคาถูก แยกตามประเภทสินค้า หรือตามชื่องานอีเวนต์/ชื่อศิลปิน
* **Stock Countdown:** ตัวนับจำนวนสินค้าที่เหลืออยู่จริงในคลังของผู้ขายรายนั้น ๆ 

### 3. ระบบความปลอดภัยและตัวกลาง (Escrow & Trust System)
* **Hold Payment:** ระบบจะเก็บเงินค่าสินค้าไว้ที่ตัวกลางก่อน เมื่อผู้ขายจัดส่งของและผู้ซื้อกดยืนยันได้รับสินค้า (หรือตามเวลาที่กำหนด) ระบบจึงจะโอนเงินให้ผู้ขาย

## 🛠️ เทคโนโลยีที่ใช้ (Tech Stack)

| ส่วนของระบบ | เทคโนโลยีที่แนะนำ | เหตุผล |
| :--- | :--- | :--- |
| **Frontend** | React (Vite) / Tailwind CSS | ทำหน้าตา UI ที่ลื่นไหล แสดงผลแถบสถานะ (Progress Bar) ได้ทันที |
| **Backend** | Node.js (Express) | ทำระบบอัปเดตยอดคงเหลือและยอดสะสมผ่าน WebSockets ได้ดี |
| **Database** | PostgreSQL | จัดการสถานะออเดอร์ การตัดสต็อก และความถูกต้องของเม็ดเงิน |
